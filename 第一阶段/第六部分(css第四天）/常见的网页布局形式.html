1.清除浮动

1.1清楚浮动原因

清除浮动的本质是：清除浮动的本质是清除浮动元素脱离标准流造成的影响

1.1.1由于父级盒子很多情况下，不方便给高度，但是子盒子又不占有位置，最后父级盒子高度为0时，就会影响下面的标准流盒子。

1.1.2由于浮动元素不再占用原文档流的位置，所以它会对后面的元素排版产生影响

1.1.3如果父盒子本身有高度，则不需要清除浮动

1.1.4清除浮动之后，

父级就会根据浮动的子盒子自动检测高度。

父级有了高度，就不会影响下面的标准流了

简单来说：要清除浮动需满足1.父级没有高度，2.子盒子浮动，3.影响下面布局

3．父级添加after伪元素(高级的清除方法)

after伪元素会在盒子后面新增个空盒子，然后后面就相当于隔墙法，不同的是after伪元素是在CSS上新增的盒子

代码： .clearfix:after {content : "";display: block;height: 0; clear: both; visibility: hidden; }

因为IE6、7浏览器不支持after和before这种为元素，所以要设置个*zoom:1来兼容

.clearfix { *zoom: 1; }

目前直接调用代码就行

注意，clearfix要加在父元素上

优点:没有增加标签，结构更简单

缺点∶照顾低版本浏览器

4．父级添加双伪元素(高级的清除方法)

.clearfix:before, .clearfix:after { content : ""; display:table; } .clearfix:after { clear:both; } .clearfix { *zoom: 1; }

优点:代码更简洁

缺点∶照顾低版本浏览器
